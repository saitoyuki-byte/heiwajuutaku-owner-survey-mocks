# 平和住宅情報センター様 アンケート管理ダッシュボード 打ち合わせ用メモ

## 目的

フォーム回答の元データをスタッフが直接編集しない運用に切り替え、回答確認、対応状況管理、スタッフコメント記録を専用ダッシュボードで行う構成を検討します。

## 打ち合わせで見せるもの

- `index.html`
  - 集計サマリー
  - 回答者一覧
  - 回答者ごとの詳細
  - スタッフコメント追加
  - コメント履歴
  - 要件確認メモ

## 会議で確認したいこと

- スタッフは全回答を見てよいか、担当分だけにするか
- コメント欄は1つでよいか、用途別に分けるか
- 対応ステータスは何種類必要か
- コメント履歴を残す必要があるか
- 誰がいつ変更したかを記録する必要があるか
- 通知メールを送る条件
- 集計画面で見たい指標
- 本番スプレッドシートの列名と回答項目

## 推奨構成

- `RawResponses`
  - フォーム回答の元データ
  - 管理者以外は編集しない
- `StaffComments`
  - スタッフコメント、対応状況、担当者、更新日時
- `AuditLog`
  - 更新履歴
- `Settings`
  - スタッフ権限、通知先、ステータス一覧

## 本番化の考え方

Google Apps ScriptのWebアプリとして実装し、スタッフはダッシュボード画面から閲覧とコメント登録を行います。
スプレッドシートはデータ保存先として使い、スタッフが直接編集しない設計にします。
